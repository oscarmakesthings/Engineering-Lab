# Lab Operating System

This workspace turns coursework into a repeatable pipeline:

`research -> experiments -> projects -> writing`

## Structure

- `research/`: topic notes, open questions, and source tracking
- `experiments/`: quick tests used to validate ideas and build intuition
- `projects/`: larger implementations that grow out of successful experiments
- `writing/`: articles, tutorials, and publication drafts
- `assets/`: diagrams, screenshots, animations, and other media
- `templates/`: reusable templates for each stage of the workflow

## Suggested workflow

1. Start in `research/` with a class concept.
2. Capture key ideas, systems examples, and unanswered questions.
3. Build a focused test in `experiments/`.
4. Promote useful results into `projects/`.
5. Publish the outcome from `writing/` with visuals from `assets/`.

## Starter topics

The `research/`, `experiments/`, and `projects/` folders already include example topic directories you can expand or rename as your degree evolves.
